# 项目开发规则 (Project Rules)

## 1. 语言偏好
- **强制中文**：所有的对话、代码注释、文档说明、Git提交信息必须使用简体中文。
- **例外**：代码中的关键字、标准库函数名、专有名词（如 React, Vue 等）保留英文。

## 2. 新手友好模式
- **解释优先**：在引入新概念或修改复杂逻辑时，必须先用通俗易懂的语言解释。
- **步骤拆解**：将复杂任务拆解为微小的、可执行的步骤。
- **安全第一**：涉及删除文件、重构核心逻辑等高风险操作前，必须进行二次确认。

## 3. 代码规范
- **注释**：关键逻辑必须添加中文注释。
- **命名**：变量和函数名使用英文（驼峰命名法），但在注释中说明其含义。
